HTML Imports

<link rel="import" id="deluxe-counter-import" href="src/deluxe-counter.html"
    onload="console.log('hooray!)" onerror="console.log('uh-oh!)">

Accessing the Import Document

var link = document.querySelector("#deluxe-counter-import").import;

Referencing the Calling Document

var thisDoc = document.currentScript.ownerDocument;

HTML Imports

  • Imports are only loaded once, no matter how many times they are referenced

  • Can import entire apps or modules

  • Links to external sites must be CORS-enabled

  • Can contain HTML, CSS, JS

  • Can be nested

  • Event handlers must be declared before the imports

HTML Imports

  • Imported content is an "import document"

  • link.import is null when:

    • The browser doesn’t support HTML Imports.

    • The <link> doesn’t have rel="import".

    • The <link> has not been added to the DOM.

    • The <link> has been removed from the DOM.

    • The resource is not CORS-enabled

HTML Imports

  • Script is still executed in the context of the main window

  • Imports block rendering (unless you use async), not parsing

  • Everything except <style> elements must be added to the main page explicitly

  • document.open(), document.write(), and document.close() are not allowed inside imports

  • Firefox has no plans to support HTML imports though for now it can be enabled through the "dom.webcomponents.enabled" preference in about:config
